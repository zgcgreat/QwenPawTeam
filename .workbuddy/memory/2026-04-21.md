## 多语言标签动态切换修复

### 问题
登录后切换语言，HeaderUserMenu 的字段标签没有跟着切换，还是显示登录时的语言。

### 根因
登录时 `storeVerifiedUserInfo` 只存储了**一种语言**的标签到 sessionStorage，语言切换后 `userInfo.fieldLabels` 不会更新。

### 修复（4 语言全部存储）
1. **types.ts**: `fieldLabels` 类型从 `Record<string, string>` 改为 `{ zh?, en?, ja?, ru? }`
2. **LoginPage.tsx**: 登录时存储所有 4 种语言的标签 `storeVerifiedUserInfo(res, {zh, en, ja, ru})`
3. **userContext.ts**: 支持新格式存储和回退逻辑
4. **HeaderUserMenu.tsx**: 新增 `getCurrentLabels()` 根据 `localStorage.language` 动态选择标签源

### password 字段
- `tenant.json` 补全了 4 语言的 password/passwordPlaceholder/passwordRequired 翻译
- 后端 `constants.py` 的 `_DEFAULT_LABELS_*` 也加入了 password 映射
- 前端 `LoginPage.tsx` 的 `DEFAULT_LABELS_*` 同步加入 password

### 数据流（修复后）
1. 登录页从 `/auth/status` 获取 4 语言标签 → `mergeLabels(DEFAULT_*, backend_labels)` 合并后存入组件 state
2. 登录成功 → `storeVerifiedUserInfo(res, {zh, en, ja, ru})` 存储 4 语言标签到 sessionStorage
3. 页面刷新 → AuthGuard 调用 `storeVerifiedUserInfo(data)` 不传标签 → 保留已有标签
4. HeaderUserMenu 每次渲染时从 `localStorage.language` 读取当前语言 → 选择对应标签源

### 根本问题与修复
- **后端 `_parse_field_labels`**: JA/RU 不再自动生成英文 fallback（`auto_fallback=False`），避免日文/俄文环境下显示英文名
- **前端 `mergeLabels`**: 后端标签与前端默认标签合并，而非 `||` 替换。解决了后端返回空 `{}` 时不触发 fallback 的问题
- **前后端默认标签**: 都从只有 `username/password` 扩展到包含 `orgId/deptId/userId/tenantId/companyId` 的 4 语言完整映射
- **`getFieldLabel` 优先级**: 后端标签(当前语言) → i18n → 自动生成。确保自定义字段优先走后端标签
