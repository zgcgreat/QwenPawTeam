## 前端 AuthGuard 认证链重构

### 第一次改动（2 级链，有 bug）
将 `MtAuthGuard` 从 3 级简化为 2 级：SSO cookie → init-workspace → 放行；无 SSO → 跳登录页。
- **Bug**：`/auth/login` 成功后 token 存 localStorage，但 AuthGuard 只查 SSO cookie → 找不到 → 回到登录页 → **死循环**

### 第二次改动（修复，3 级链）
恢复为 3 级认证链，同时保持逻辑清晰：
1. **SSO cookie** (getSsoToken) → init-workspace → 放行
2. **localStorage token**（来自 /auth/login）→ /auth/verify → 放行
3. **无凭证** → 跳登录页

### 改动文件（均在 `console/src/multi_tenant/` 内，零侵入上游代码）
1. **AuthGuard.tsx** — 3 级认证链：SSO cookie → localStorage token verify → 无凭证跳登录
2. **requestExtension.ts** — 移除 `getCookieValue` 依赖和 SSO cookie 静默跳过逻辑；401 统一清状态+跳登录页

### 未改动文件
- `App.tsx`、`request.ts`、`api/authHeaders.ts`、`LoginPage.tsx`、`authApi.ts`、`authHeaders.ts` 均无修改
- 非多租户模式 (`MULTI_TENANT_ENABLED=false`) 完全不受影响
